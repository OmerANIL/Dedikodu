# Dedikodu App

## Overview
Location-based gossip sharing platform (Turkish: "Dedikodu"). Full-stack with NestJS backend + React Native Expo frontend.

## Tech Stack
- **Backend**: NestJS + Prisma + PostgreSQL
- **Frontend**: React Native (Expo SDK 54), TypeScript, expo-router, react-native-paper
- **Auth**: JWT stored in AsyncStorage, Bearer token in Authorization header

## Frontend Architecture

### Navigation (expo-router file-based)
```
app/
  _layout.tsx          — Root: ThemeProvider > AuthProvider > PaperProvider > Stack
  index.tsx            — Redirect based on auth state
  +not-found.tsx       — 404 page
  auth/
    _layout.tsx        — Guard: authenticated → redirect to /tabs/home
    login.tsx          — Email + password login
    register.tsx       — Full registration form
    verify-email.tsx   — 6-digit code verification
  tabs/
    _layout.tsx        — Guard: !authenticated → redirect to /auth/login; Bottom tabs
    home.tsx           — Gossip feed with location filter, reactions, infinite scroll
    write.tsx          — Create gossip (gold/platinum gate, email verification gate)
    moderation.tsx     — Pending gossip review (platinum gate, tab hidden for non-platinum)
    profile.tsx        — User info, theme toggle, logout, admin access
  subscription.tsx     — Subscription plans (modal)
  admin/
    _layout.tsx        — Guard: superuser only
    index.tsx          — Admin dashboard with statistics
    users.tsx          — User management with search + subscription level change
```

### Shared Components
- **GradientButton**: Orange-red gradient CTA with loading/disabled states
- **AppInput**: Styled text input with floating label
- **GossipCard**: Gossip display with reactions (approve/disapprove)
- **LocationFilterBar**: Cascading province→district→neighborhood filter
- **LocationPicker**: Required 3-step location selector for gossip creation
- **ErrorBoundary**: Class-based error boundary for crash prevention

### Contexts
- **AuthContext**: user, isAuthenticated, isLoading, login, signup, logout, refreshUser
- **ThemeContext**: isDark, colors, toggleTheme (persisted in AsyncStorage)

### Theme
- Dark mode default (#0D0D0D backgrounds)
- Light mode (#F5F2EB backgrounds)
- Primary: #F97316 (orange), Accent: #EF4444 (red)
- Approve: #10B981, Disapprove: #EF4444
- Subscription badges: Temel (gray), Gold (amber), Platin (purple)

### API Endpoints Used
- POST /api/signup, POST /api/auth/login, GET /api/auth/me
- POST /api/auth/verify-email, POST /api/auth/send-verification
- GET /api/gossips (with location filters), POST /api/gossips
- POST /api/gossips/:id/react, DELETE /api/gossips/:id/react
- GET /api/gossips/pending, PATCH /api/gossips/:id/moderate
- GET /api/admin/stats, GET /api/admin/users, PATCH /api/admin/users/:userId
- GET /api/provinces, GET /api/provinces/:id/districts, GET /api/districts/:id/neighborhoods

### Key Design Decisions
- All UI text in Turkish
- Moderation tab hidden (href: null) for non-platinum users
- Write tab visible but shows gate message for basic users
- No self-subscription endpoint in backend; subscription screen uses admin endpoint for superusers, shows info message for regular users
- Relative dates in Turkish format
- Dark/light theme persisted via AsyncStorage
